# Анализ Импортов в `src/`

## Резюме
Проведен анализ всех Python файлов в директории `src/` на предмет циклических зависимостей.

## Результаты

### Структура проекта
Проект состоит из 50 Python файлов, организованных в следующие модули:
- `api/` - API endpoints и зависимости
- `bot/` - Telegram bot handlers и middlewares
- `core/` - Логирование и middleware
- `database/` - Модели, подключение, repository, UoW
- `schemas/` - Pydantic схемы
- `services/` - Бизнес-логика
- `workers/` - Фоновые задачи

### Циклические зависимости
**Не обнаружено** явных циклических зависимостей.

### Рекомендации
Текущая архитектура следует принципу разделения ответственности:
- `database/models.py` - определяет модели SQLAlchemy
- `schemas/` - определяет Pydantic схемы для валидации
- `services/` - использует модели и схемы
- `api/` - использует сервисы

Такая структура предотвращает циклические импорты.
